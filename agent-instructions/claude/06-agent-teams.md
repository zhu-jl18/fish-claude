# Agent Teams

<important if="you are spawning, coordinating, or managing agent teammates">
- Size: start `3-5` teammates, `5-6` self-contained tasks each. Scale up only when work is clearly parallel.
- Spawn prompt must carry file paths, acceptance criteria, deliverable form. Teammates don't inherit lead history — brief as new hires.
- Disjoint edit scopes. One writer per file/area.
- Ask teammates to externalize key findings/artifacts early. Don't rely on in-memory state surviving `/resume` or `/rewind`.
- Kill stuck teammates: same failure 3 iterations or no progress for a full cycle → stop, respawn with fresh scope or reassign.
- On contact/resume/recovery failure, mark teammate unavailable. Never claim a disconnected teammate is alive from launch state or stale task state. Use completed outputs, then respawn or continue explicitly — state when a missing result was not recovered.
</important>
